Guia de Acesso ao Repositório

Documento de apoio para navegação da documentação e execução do projeto.

1. Link do Repositório

Repositório Git:

https://github.com/anderson-napoleao/fiap-tech-challenge-5.git

2. Onde estão as documentações

  • README.md: visão geral da solução, arquitetura, endpoints, execução local, testes, observabilidade e roteiro de demonstração.
  • DOCKER.md: operação do ambiente com Docker Compose, serviços, troubleshooting e validações de CDC.
  • tests-integracao-sistema/README.md: estratégia e execução dos testes cross-module (integração entre serviços).
  • docs/openapi/: especificações OpenAPI/Swagger de cada serviço (servico-identidade, servico-usuario, servico-encomenda, servico-notificacao).
  • infra/debezium/: configuração dos conectores CDC do padrão Outbox (Debezium/Kafka Connect).

3. Onde estão os comandos para executar

Os comandos principais estão no README.md (seções 6 e 7) e no DOCKER.md. Resumo:

Subir ambiente com Docker

docker compose up -d --build
docker compose ps

Validar conectores CDC

curl http://localhost:8085/connectors
curl http://localhost:8085/connectors/encomenda-outbox-connector/status
curl http://localhost:8085/connectors/notificacao-outbox-connector/status

Testes padrão

mvn test

Testes de integração cross-module

mvn -pl tests-integracao-sistema -am test -Dtestcontainers.enabled=true -Dtest=FluxoCadastroUsuarioCrossModuleTest -Dsurefire.failIfNoSpecifiedTests=false
mvn -pl tests-integracao-sistema -am test -Dtestcontainers.enabled=true -Dtest=FluxoEncomendaNotificacaoRetiradaCrossModuleTest -Dsurefire.failIfNoSpecifiedTests=false

Quality gate (PowerShell)

mvn -Pquality '-DskipTests=true' '-Ddependency-check.skip=true' verify
